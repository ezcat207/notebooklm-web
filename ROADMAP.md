## 📊 功能详细分析

### P0-1: 批量导入 URLs（最高频，立即做）

**用户场景：**
```
内容创作者：每周 20+ 视频/文章链接
研究生：100+ 论文 URL
```

**为什么优先级最高？**
- ✅ **使用频次最高** - 每次开始新项目都需要
- ✅ **痛点明确** - 逐个添加 URL 极其繁琐
- ✅ **实现简单** - 我们已有 `addUrlSource` 函数
- ✅ **差异化不大** - NLMTools 已有，必须有的基础功能

**实现方案：**
```
输入框：粘贴多行 URLs（逗号/换行分隔）
          ↓
后台批量调用 ADD_SOURCE RPC
          ↓
进度条显示：10/50 已导入
          ↓
完成：显示成功/失败列表
```

**开发时间：** 1-2 天

---

### P0-2: 批量生成 Studio 内容（核心竞争力）

**用户场景：**
```
内容创作者：10 个 Notebooks，每个生成 Audio + Video + Slides
学生：期末复习，批量生成所有课程的 Quiz + Flashcards
```

**为什么是核心竞争力？**
- ✅ **NLMTools Pro 的核心卖点**（$29/年）
- ✅ **高频需求** - 用户每次整理完 Sources 就需要生成
- ✅ **我们已实现** - 已有 `createStudio` 函数和所有 artifact types
- ✅ **差异化机会** - 我们可以做得更强（进度追踪、失败重试）

**实现方案：**
```
选择多个 Notebooks
          ↓
选择生成类型（Audio/Video/Quiz/Slides/Report/...）
          ↓
并行调用 CREATE_STUDIO RPC（限制并发数避免被限流）
          ↓
实时进度：Notebook A - Audio (完成) / Video (生成中...)
          ↓
完成：自动下载所有生成内容
```

**技术挑战：**
- 并发控制（避免触发 Google 限流）
- 进度追踪（POLL_STUDIO 轮询）
- 失败重试逻辑

**开发时间：** 3-5 天

---

### P0-3: 批量下载所有内容（高频，中等难度）

**用户场景：**
```
学生：下载所有课程的 MP3/PDF 备份
研究者：导出所有 Reports/Data Tables 到本地
```

**为什么重要？**
- ✅ **备份需求** - 用户不想数据只在 Google 服务器
- ✅ **离线使用** - Audio Overview 离线收听
- ✅ **NLMTools 只做了音频** - 我们可以做全部类型

**实现方案：**
```
选择 Notebook
          ↓
列出所有 Studio 内容（Audio/Video/Slides/PDF...）
          ↓
批量下载（Chrome downloads API）
          ↓
打包成 ZIP（可选）
```

**开发时间：** 2-3 天

---

### P1-1: 批量删除 Notebooks/Sources（高需求，简单）

**用户场景：**
```
学生：学期结束，删除 50+ 个过时课程 Notebooks
研究者：清理失败的测试 Notebooks
```

**为什么是 P1 而不是 P0？**
- ✅ **需求强烈**（Reddit 25+ 投票）
- ⚠️ **但使用频次低于导入/生成** - 只在清理时用
- ✅ **实现简单** - 已有 `deleteNotebook` 函数

**实现方案：**
```
Notebook 列表 → 多选框 → 批量删除按钮 → 确认对话框 → 并行删除
```

**开发时间：** 1 天

---

### P1-2: 去重检测（中等需求，中等难度）

**用户场景：**
```
用户：不小心重复添加了同一个 URL
希望：自动检测并提示删除重复项
```

**实现方案：**
```
检测逻辑：
- URL 完全匹配
- 标题相似度 > 90%
- 文件哈希匹配（对于上传的文件）

UI：
在 Source 列表显示 "Duplicate" 标签
批量删除重复项按钮
```

**开发时间：** 2-3 天

## 🚀 Phase 2 功能（MVP 之后）

| 优先级 | 功能 | 为什么不是 MVP |
|--------|------|--------------|
| **P2** | **文件夹/标签组织** | 实现复杂，需要新的 UI 设计 |

| **P3** | **多账户支持** | 复杂的认证切换逻辑 |

## 💰 商业化对比

### NLMTools 的成功经验

| 指标 | 数据 |
|------|------|
| 用户数 | 80,000+ |
| 付费定价 | $29/年 或 $59 终身 |
| 年收入 | $50,000-$80,000 |
| 核心卖点 | 批量 Studio 生成（Pro 独占） + 无干扰界面 |
| 转化率 | ~3% |

### 我们的策略建议

**免费版（获取用户）：**
- ✅ 批量导入 URLs（基础功能）
- ✅ 批量删除（基础功能）
- ⚠️ 批量生成 Studio（限制：每次最多 3 个 Notebooks）

**付费版（$29/年 或 $59 终身）：**
- ✅ **无限批量生成 Studio**
- ✅ **批量下载所有内容**
- ✅ **去重检测 + 自动清理**
- ✅ **优先支持 + 功能抢先体验**
